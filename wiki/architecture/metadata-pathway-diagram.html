Livrarr Metadata Pathway

All normal add paths converge through WorkService::add. Existing works re-enter through refresh/retry. The pipeline locks the work identity, gathers provider metadata, validates identity, merges fields, downloads covers, and retags files. Open the companion pseudocode.

Validation And Merge

Identity Validation

Provider payloads are compared to the locked title/author anchor. Bad matches become rejected outcomes.

Current Goodreads Risk

Search fallback is LLM-gated and lacks deterministic junk filtering before selection.

MergeEngine

Chooses field winners by provider priority while preserving user-owned fields and manual covers.

DB Apply

apply_enrichment_merge writes fields, provenance, external IDs, status, and merge generation.

Outputs

Work Metadata

Canonical fields live on works, with field-level provenance in work_metadata_provenance.

Cover Cache

If the merged work has cover_url and no manual cover, Livrarr downloads the image to the cover cache.

File Tags

Existing library items are retagged from the post-enrichment work metadata.

Conflict State

If all successful provider payloads are rejected, the work becomes conflict. No metadata or cover URL is merged.

Manual Add From Search

User searches, selects a metadata candidate, then POSTs a comparatively rich add request with selected title, author, keys, cover URL, language, and detail URL.

Manual Import

Scanner parses local files and OpenLibrary matches, but the final import request currently carries only path, OL key, title, author, delete flag, and language.

Bulk Sources

Readarr, list import, and author monitor create works in batches. Readarr is the richest source because it injects source provider data.

Existing Works

Refresh, bulk refresh, and background retry skip work creation and re-enter at enrichment for an existing work ID.

Important Detail

  • Manual import often starts with sparse seed data, so provider matching has to do more work.
  • Readarr import starts with richer SourceProviderData, including covers and identifiers.
  • Goodreads direct detail pages can work when Livrarr already has a gr_key.
  • Goodreads search is currently the fragile part: it can be WAF-challenged and its candidate selection is too LLM-dependent.
